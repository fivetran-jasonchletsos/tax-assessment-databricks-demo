# Allegheny County Tax Assessment Portal

Public-facing portal for the Allegheny County property tax assessment data,
deployed as a static SPA on GitHub Pages and refreshed daily from Databricks.

**Live site:** https://fivetran-jasonchletsos.github.io/tax-assessment-databricks-demo/

## Architecture

```
Fivetran custom connectors  →  Databricks Unity Catalog  →  dbt marts
        (WPRDC + Allegheny County)       (raw → staging → marts)

                                          │
                                          ▼

                       scripts/build_snapshot.py
                  (GitHub Actions, daily 06:17 UTC)

                                          │
                                          ▼

           tax-assessment-app/frontend/public/data/*.json  ──▶  GitHub Pages
                          (React + Vite + Recharts + Leaflet)
```

The deployed site is fully static. It reads a JSON snapshot of the dbt marts
that is regenerated by `.github/workflows/refresh-data.yml`.

## Repo layout

| Path | Purpose |
| --- | --- |
| `tax-assessment-app/frontend/` | React + TypeScript SPA (Vite) |
| `tax-assessment-app/backend/`  | FastAPI service for local dev with Databricks |
| `tax-assessment-app/dbt_project/` | dbt models that build the marts |
| `tax-assessment-app/fivetran_connector/` | Custom Fivetran SDK connector |
| `tax-assessment-app/scripts/build_snapshot.py` | Snapshot extractor used by CI |
| `.github/workflows/deploy.yml` | Builds and publishes the SPA to Pages |
| `.github/workflows/refresh-data.yml` | Daily Databricks pull, commits JSON |

## Enabling the live data refresh

Add these repository secrets at `Settings → Secrets and variables → Actions`:

| Secret | Example |
| --- | --- |
| `DATABRICKS_HOST` | `dbc-c48d38b1-67f3.cloud.databricks.com` |
| `DATABRICKS_HTTP_PATH` | `/sql/1.0/warehouses/<id>` |
| `DATABRICKS_TOKEN` | personal access token |

Optional repository variables:

| Variable | Default |
| --- | --- |
| `DATABRICKS_CATALOG` | `jason_chletsos_allegheny_tax` |
| `DATABRICKS_MARTS_SCHEMA` | `jason_chletsos_marts` |
| `SNAPSHOT_MAX_PARCELS` | `500` |

Run the workflow manually once from the Actions tab to pull the first
snapshot. Subsequent runs happen daily at 06:17 UTC and only commit when
the snapshot actually changed.

## Local development

```bash
cd tax-assessment-app/frontend
npm install
VITE_BASE=/ npm run dev   # http://localhost:5173/
```

To regenerate the local snapshot against Databricks:

```bash
cd tax-assessment-app
DATABRICKS_HOST=... DATABRICKS_HTTP_PATH=... DATABRICKS_TOKEN=... \
    python3 scripts/build_snapshot.py
```

Without those env vars the script writes a curated sample so the site
always renders.
